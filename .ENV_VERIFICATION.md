## Why Still 503 Error?

**Your .env is configured correctly.** The 503 error is likely because:

### Issue 1: Node.js Application Not Running ⚠️

**In cPanel → Node.js Application Manager:**

1. **Look for your app** (likely named "oes-backend" or similar)
2. **Check the status:**
   - 🟢 Green = "Running" ✅
   - 🔴 Red = "Stopped" ❌
   - ⚪ Grey = "Error" ❌

3. **If NOT running:**
   - Click **"Restart"** button
   - Wait **30-45 seconds**
   - Refresh your browser (Ctrl+F5)

---

### Issue 2: Startup Error in Application

**Check Application Logs in cPanel:**

1. Go to **Node.js Application Manager**
2. Click **"View Logs"** or **"Open Logs"**
3. **Look for errors like:**

```
Error: Cannot find module 'nodemailer'
```
**Solution:** Application needs npm install

```
Error: connect ECONNREFUSED 127.0.0.1:3306
```
**Solution:** MySQL not running or wrong host

```
Error: ER_ACCESS_DENIED_FOR_USER
```
**Solution:** Database credentials wrong (but yours look correct)

---

### Issue 3: Missing npm Dependencies

**In cPanel → Terminal:**

```bash
cd /home/username/public_html/oes-backend
npm install
npm start
```

**You should see:**
```
[✅ SESSION] File-based session store configured
[✅ CORS] Allowing requests from: https://oes.freshmilkstraightfromsource.com
[✅ EMAIL] Email service initialized (or ⚠️ if SMTP issue)
Backend running on http://localhost:5000
```

If you see this, the app is working! The 503 might just be from before the restart.

## Quick Diagnostic Test

**Run these commands in cPanel Terminal:**

### Test 1: Check Node.js Running
```bash
ps aux | grep node
```
**Should show:** `/usr/local/bin/node /path/to/src/server.js`  
**If nothing:** Application not running → restart in App Manager

### Test 2: Check Database Connection
```bash
mysql -h localhost -u freshmil_oesuser -p
# Password: Reddys4512@

# Then type:
USE freshmil_oes;
SELECT COUNT(*) FROM users;
exit;
```
**Should show:** Number of users without error  
**If error:** Database credentials issue

### Test 3: Check Port Listening
```bash
netstat -tulpn | grep 5000
# or: ss -tulpn | grep 5000
```
**Should show:** `LISTEN` on `127.0.0.1:5000`  
**If nothing:** Node not running on port 5000

### Test 4: Verify Database Columns Exist
```bash
mysql -h localhost -u freshmil_oesuser -p freshmil_oes
# Password: Reddys4512@

# Check columns:
DESCRIBE users;
# Should show: reset_token, reset_token_expires columns
```

**If missing columns**, run in phpMyAdmin:
```sql
USE freshmil_oes;
ALTER TABLE users ADD COLUMN reset_token VARCHAR(255) NULL;
ALTER TABLE users ADD COLUMN reset_token_expires TIMESTAMP NULL;
```
